# 🚀 Vercelでデプロイする手順（GitHubコミット後）

## ステップ1: Vercelに登録（2分）

1. **ブラウザでVercelを開く**
   - https://vercel.com にアクセス

2. **サインアップ**
   - 「Sign Up」ボタンをクリック
   - 「Continue with GitHub」をクリック
   - GitHubアカウントでログイン
   - 「Authorize Vercel」をクリック（GitHubの認証を許可）

## ステップ2: プロジェクトをインポート（3分）

1. **ダッシュボードでプロジェクトを作成**
   - 「Add New...」→「Project」をクリック
   - または「New Project」ボタンをクリック

2. **GitHubリポジトリを選択**
   - 「Import Git Repository」セクションで
   - あなたが作成したリポジトリ（例: `soccer-standings`）を選択
   - 「Import」をクリック

3. **プロジェクト設定**
   - **Project Name**: そのままでOK（または変更可）
   - **Framework Preset**: **Other** を選択
   - **Root Directory**: `./` （そのまま）
   - **Build Command**: （空欄のまま）
   - **Output Directory**: （空欄のまま）
   - **Install Command**: `npm install` （そのまま）

4. **環境変数**
   - 今回は設定不要（そのまま）

## ステップ3: デプロイ（2分）

1. **デプロイを開始**
   - 下部の「Deploy」ボタンをクリック

2. **待機**
   - 「Building...」と表示されます
   - 2〜3分かかります
   - 進行状況が表示されます

3. **完了！**
   - 「Congratulations!」と表示されたら成功です
   - 「Visit」ボタンをクリックすると、あなたのアプリが開きます

## 🎉 完成！

## 📝 今後の更新方法

### ファイルを更新したら

1. **GitHub Desktopでコミット**
   - 変更を確認
   - コミットメッセージを入力
   - 「Commit to main」→「Push origin」をクリック

2. **自動デプロイ**
   - Vercelが自動的に変更を検知
   - 数分で新しいバージョンが公開されます
   - 手動で何かする必要はありません！

## ❓ トラブルシューティング

### デプロイが失敗する場合

1. **Vercelのダッシュボードを確認**
   - 「Deployments」タブを開く
   - 失敗したデプロイをクリック
   - 「Logs」でエラーを確認

2. **よくあるエラー**
   - `npm install` が失敗 → `package.json`を確認
   - ポートエラー → `server.js`の`PORT`設定を確認（既に修正済み）

### アプリが開かない場合

1. **数分待つ**
   - 初回デプロイは少し時間がかかることがあります

2. **ブラウザのキャッシュをクリア**
   - 「Cmd + Shift + R」で強制リロード

## 💡 便利な機能

### カスタムドメイン（オプション）

1. Vercelのダッシュボードで「Settings」→「Domains」
2. ドメイン名を入力
3. DNS設定を案内に従って設定

### プレビューデプロイ

- プルリクエストを作成すると、自動的にプレビューURLが作成されます
- 本番環境に影響を与えずにテストできます
