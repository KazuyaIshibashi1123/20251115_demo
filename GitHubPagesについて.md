# GitHub Pagesについて

## ❌ GitHub Pagesの制限

**GitHub Pagesは静的サイトのみ対応**しています。つまり：
- ✅ HTML、CSS、JavaScript（クライアントサイド）は使える
- ❌ Node.jsサーバー（`server.js`）は実行できない
- ❌ サーバーサイドのスクレイピング機能は使えない

このアプリは`server.js`でスクレイピングを行っているため、**GitHub Pagesだけでは動作しません**。

## ✅ 解決方法

### 方法1: Vercelを使う（推奨・最も簡単）

GitHub Pagesの代わりに**Vercel**を使えば、Node.jsサーバーも実行できます。

**メリット：**
- ✅ 無料
- ✅ Node.jsサーバーが使える
- ✅ GitHubと連携して自動デプロイ
- ✅ 設定が簡単

**手順：**
1. https://vercel.com にアクセス
2. GitHubアカウントでログイン
3. リポジトリを選択してデプロイ

詳細は `Vercelデプロイ手順.md` を参照してください。

### 方法2: GitHub Pages + Vercel Functions（上級者向け）

GitHub Pagesで静的ファイルをホスティングし、API部分だけVercel Functionsを使う方法です。

**手順：**
1. `server.js`をVercel Functions用に分割
2. 静的ファイルをGitHub Pagesにデプロイ
3. API部分をVercel Functionsにデプロイ

この方法は複雑なので、**方法1（Vercel全体）を推奨**します。

### 方法3: 完全にクライアントサイドに変更（制限あり）

サーバーを使わず、ブラウザから直接スクレイピングする方法です。

**問題点：**
- ❌ CORS（クロスオリジン）エラーが発生する可能性が高い
- ❌ 多くのサイトはブラウザからの直接アクセスをブロック
- ❌ セキュリティ上の制限が多い

**結論：この方法は実用的ではありません。**

## 🎯 推奨：Vercelを使う

GitHub Pagesの代わりに**Vercel**を使うことを強く推奨します。

### Vercelのメリット

1. **無料プランで十分**
   - 個人利用なら無料で使えます
   - サーバーレス関数も無料で使えます

2. **GitHubと連携**
   - GitHubにプッシュすると自動デプロイ
   - GitHub Pagesと同じように使えます

3. **簡単な設定**
   - `vercel.json`ファイルが既に準備済み
   - ほぼそのままでデプロイ可能

4. **カスタムドメイン**
   - 無料プランでもカスタムドメインを設定可能
